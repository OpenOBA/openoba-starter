# ERA-Chat 运营指南

> 怎么用自然语言运营眼镜 ERP——查数据、调价、促销、导入导出、生成报表

## 这份指南适合谁

- 不想记菜单、不想学 ERP 操作的运营/业务人员
- 需要快速出数据报表的运营主管
- 负责批量导入导出数据的实施/客服

## 前置准备

1. OpenOBA Starter 已启动，能登录
2. 在 **ERA-Chat → 设置 → API Key** 配置了 LLM Key
3. 你的账号角色有对应业务模块的读/写权限（不需要 `canEditCode`）

> 💡 **运营角色不需要 `canEditCode`**。运营人员用 `query-erp-data` / `erdl-crud` / `csv-exporter` 等 skill，这些 skill 不需要改代码权限。详见 [安全架构](../security/security-architecture.md)。

## 核心心智模型

在 ERA-Chat 里运营，你把 AI 当成一个**懂业务、会查数据、能动手的运营助理**。你的工作是：

1. **说清楚要什么**——查什么、改什么、导出什么
2. **看 AI 返回的结果**——表格、图表、文件
3. **确认动作**——涉及写数据时，AI 会先给方案让你确认

AI 能帮你做（你不用教它）：
- 查 ERP 真实业务数据（库存/订单/客户/商品）
- 读写数据库（受权限白名单约束，危险操作要确认）
- 导入 CSV/Excel（三步流水线：分析 → 映射 → 执行）
- 导出 CSV
- 数据分析（统计/排行/趋势）
- 抓取网页（竞品价格/行业资讯）
- 生成报表（可下载的 Deliverable）

## 场景一：查数据

### 简单查询

```
查一下库存低于 10 的镜架
```

AI 会调用 `query-erp-data` skill，返回一个表格：

```
| SKU | 名称 | 库存 | 预警阈值 |
|-----|------|------|---------|
| SKU-A12345 | 钛架-金 | 3 | 10 |
| SKU-B67890 | 板架-黑 | 7 | 10 |
...
```

### 多条件查询

```
查本月退货率最高的 5 个 SKU，包含退货数和订单数
```

AI 会：
1. 用 `query-erp-data` 拉 AfterSales 和 Orders 数据
2. 用 `data-analyzer` 算退货率（退货数/订单数）
3. 返回排行表 + 分析建议

### 自然语言追问

```
上面排第一的 SKU-A12345，近 3 个月销量趋势怎么样？
```

AI 会在上一轮上下文基础上继续分析，不需要你重复说 SKU 号。

## 场景二：调价 / 调库存

### 批量调价

```
把所有"钛架"分类下的 SKU 价格上调 10%，
但不要动 VIP 专属的 3 款（SKU-V001/V002/V003）。
先给我看调价清单，我确认后再执行。
```

AI 会：
1. 查询符合条件的 SKU
2. **先返回调价清单**（不直接改）：

```
拟调价清单（共 27 款，已排除 3 款 VIP 专属）：
| SKU | 原价 | 新价 | 涨幅 |
|-----|------|------|------|
| SKU-A12345 | ¥299 | ¥329 | +10% |
...
```

3. 你说"确认执行"后，AI 用 `erdl-crud` 的 update 操作批量改价
4. 返回执行结果（成功数/失败数）

> ⚠️ **安全提示**：涉及写数据的操作，AI 默认会先出方案让你确认。这是 SOUL 铁律约束的——"写入数据库前必须验证权限并获得人类确认"。

### 库存盘点调整

```
我刚盘完 A 区仓库，SKU-A12345 实际库存是 5（系统显示 3），
SKU-A12346 实际是 0（系统显示 2），
帮我生成一张盘点单并调整库存。
```

AI 会用 `erdl-crud` 在 `inventory_document` 表建一张盘点单，自动生成库存调整事务。

## 场景三：导入数据

### 上传 CSV 导入

**需求**：供应商发来一份报价单 CSV，要导入系统。

**操作**：
1. 在 ERA-Chat 输入框旁点上传按钮，选 CSV 文件
2. 输入：

```
把这份报价单导入到供货记录表，
列对应关系你先识别一下，我确认后再写入。
```

AI 会走**三步流水线**：

**第一步：分析**（`import-analyzer`）
```
Agent (Tool): import_analyzer { file: "报价单.csv" }
Agent (Observation): 文件含 5 列：型号/品名/含税价/最小起订量/交期
                     共 142 行数据，编码 UTF-8，分隔符逗号。
```

**第二步：映射**（`import-mapper`）
```
Agent (Tool): import_mapper { ... }
Agent: 识别到字段对应关系：
       型号 → SupplyRecord.sku_code
       品名 → SupplyRecord.product_name
       含税价 → SupplyRecord.purchase_price
       最小起订量 → SupplyRecord.moq
       交期 → SupplyRecord.lead_time_days
       
       有 3 行"型号"为空，建议跳过。确认执行吗？
```

**第三步：执行**（`import-executor`）
你说"确认"后：
```
Agent (Tool): import_executor { ... }
Agent: 导入完成。成功 139 行，跳过 3 行（型号为空）。
       可在"供货记录"页面查看。明细见 Deliverable #v1。
```

### 导入新商品

```
把这份商品 CSV 导入到 SPU 表，
图片 URL 列先存起来，后面我再批量上传图片。
```

## 场景四：导出数据

### 导出 CSV

```
导出本月所有已发货订单，包含：订单号/客户名/金额/发货日期，
按发货日期倒序，导成 CSV。
```

AI 用 `csv-exporter` skill 生成文件，返回下载链接。

### 导出报表

```
生成一份 2026 年 6 月销售分析报告，包含：
1. 总销售额、订单数、客单价
2. Top 10 畅销 SKU
3. Top 5 退货 SKU
4. 按分类的销售占比
导成一份可下载的报告。
```

AI 会：
1. 用 `query-erp-data` 拉数据
2. 用 `data-analyzer` 做统计
3. 打包成 Deliverable（含 Markdown 报告 + CSV 明细）

## 场景五：客户运营

### 生日营销

```
查本月过生日的 VIP 客户，给他们发一张 9 折优惠券，
有效期 30 天。先给我看名单。
```

AI 查 Customer 表（birthday 字段 + 会员等级），返回名单，你确认后批量发券。

### 流失预警

```
分析一下近 90 天没有下单的客户，按最后下单时间排序，
导出前 100 名，我要做召回。
```

### 客户分群

```
把客户按消费金额分成 4 档：
- 钻石：累计消费 ≥ 10000
- 金：5000-9999
- 银：1000-4999
- 普通：< 1000
统计每档人数和占比，画个饼图。
```

## 场景六：订单/售后处理

### 异常订单排查

```
查最近 7 天状态卡在"待发货"超过 48 小时的订单，
列出订单号、客户、金额、卡住时长。
```

### 售后趋势

```
本月售后单按原因类型统计，哪种原因最多？
和上月比变化大吗？
```

## 场景七：抓取外部信息

### 竞品价格

```
抓一下这两个竞品页面的价格：
- https://competitor-a.com/product/123
- https://competitor-b.com/product/456
和我们同类 SKU 对比，列个价差表。
```

AI 用 `web-fetcher` 抓页面，提取价格，和系统内 SKU 对比。

### 行业资讯

```
搜一下最近"眼镜行业 2026 趋势"相关资讯，
整理 5 条要点。
```

AI 用 `web-search` 搜索，`web-fetcher` 抓详情，汇总要点。

## 常用话术模板

### 查询类

| 想要 | 话术 |
|------|------|
| 看库存 | "查一下 [品类] 库存低于 [阈值] 的" |
| 看订单 | "查 [时间范围] [状态] 的订单" |
| 看客户 | "查 [条件] 的客户，按 [字段] 排序" |
| 看排行 | "查 [时间] [指标] Top [N] 的 [对象]" |

### 操作类

| 想要 | 话术 |
|------|------|
| 调价 | "把 [范围] 的价格 [上调/下调] [比例]，先给我看清单" |
| 调库存 | "盘点了 [区域]，[SKU] 实际 [数量]，生成盘点单" |
| 发券 | "给 [客户群] 发 [折扣] 优惠券，有效期 [天数]" |
| 改状态 | "把 [范围] 的 [对象] 状态改为 [状态]" |

### 导入导出类

| 想要 | 话术 |
|------|------|
| 导入 | "把这份 [文件] 导入到 [表]，先识别列对应关系" |
| 导出 | "导出 [范围] 的 [对象]，包含 [字段]，导成 CSV" |
| 报表 | "生成 [时间] [主题] 分析报告，包含 [内容]" |

## 高级技巧

### 1. 让 AI 主动给建议

```
看下本月销售数据，给我 3 条可执行的运营建议。
```

AI 不仅出数据，还会基于数据给建议（如"SKU-A12345 退货率高，建议下架复查"）。

### 2. 多轮对话深挖

```
（第一轮）查本月退货率最高的 SKU
（第二轮）排第一的这个，看具体退货原因分布
（第三轮）原因里"质量问题"占多数，查这些退货单关联的批次
（第四轮）这批次的供货商是谁？历史退货率怎么样？
```

AI 会在上下文里记住前面的对象，不需要你重复说。

### 3. 定时任务（需开发者配合）

运营类重复任务（如"每周一发上周销售汇总"）可以让开发者用 ERA-Chat 的任务系统配置定时 Agent，自动跑并推送结果。详见 [ERA-Chat 开发指南](./development-guide.md)。

### 4. 议会模式做决策

```
@运营 @采购 @财务 这批滞销库存（3 个月未动）怎么处理？
降价清仓、退供应商、还是打包促销？各自给方案。
```

多 Agent 各从角色视角分析，最后汇总决策建议。

## 常见问题

### AI 查的数据是最新的吗？

是。`query-erp-data` 直接查 MySQL 实时数据，不是缓存。但 AI 会在一次对话内缓存查询结果（避免重复查），如果你刚改了数据，可以说"重新查一下"。

### AI 能改哪些表？

受 `erdl-crud` skill 权限白名单约束：

| 表 | 读 | 写 |
|----|----|----|
| product_spu / product_sku | ✅ | ❌（走专门接口） |
| inventory / inventory_document | ✅ | ✅ |
| orders | ✅ | ❌（走订单接口） |
| customers | ✅ | ❌（走客户接口） |
| draft（草稿） | ✅ | ✅ |
| user / role / permission | ❌ | ❌ |
| cognitive_log / agent_task | ❌ | ❌ |

涉及"不能直接写"的表，AI 会引导你去对应的表单页面操作，或用对应的业务接口。

### AI 会误操作吗？

三层防护：
1. **写操作先出方案**：SOUL 铁律要求"写入前必须获人类确认"
2. **权限白名单**：碰不到 user/role/permission 等敏感表
3. **审计日志**：每次写操作记录到 `cognitive_log`，可追溯

### 导入数据出错怎么办？

导入是三步流水线，"执行"步骤前你可以看映射关系。如果执行后发现有错：
- 导入的草稿在 `draft` 表，可以删除重导
- 已写入业务表的，找开发者用 SQL 回滚（AI 不直接执行 SQL）

### AI 能生成图表吗？

能。AI 可以返回 Markdown 表格、ASCII 图表。如果需要可视化图表，可以让它生成 ECharts 配置或导出 Excel 图表。

## 不能做什么（安全边界）

| 操作 | 能否 | 替代方案 |
|------|------|---------|
| 查业务数据 | ✅ | — |
| 改库存/草稿 | ✅（需确认） | — |
| 改商品价格 | ✅（需确认） | — |
| 改订单/客户 | ❌ | 去对应表单页面操作 |
| 改用户/角色/权限 | ❌ | 找管理员在系统管理操作 |
| 直接执行 SQL | ❌ | 让开发者通过 ERDL 或接口 |
| 删除业务数据 | ❌ | 软删除走业务接口 |
| 发送真实短信/邮件 | ❌（需配置） | 配置阿里云短信后可用 |
