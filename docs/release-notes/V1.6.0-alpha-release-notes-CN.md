# OpenOBA V1.6.0-alpha 版本更新说明

**发布日期**：2026-06-29  
**基线版本**：V1.5.1（`b4ccbc6`，已发布至 GitHub）  
**提交数**：13 commits  
**改动范围**：19 个文件，+894 / -86 行  

---

## 🔴 P0 修复（4 项）

### 1. Agent 身份兜底修复（`06da121`）
- Agent 在 `sys_agent_manifest` 表为空时不再降级为 `sub`/`L1`
- 核心 Agent（tanghaoran / main-agent）硬编码兜底为 `main`/`L4`
- 确保管理工具（file_edit / tsc_check / git_diff）始终可用

### 2. 元镜上下文注入 Agent System Prompt（`ce254c4`）
- 新增 `MirrorKnowledgeReader` — 从 `knowledge/` 目录动态读取实体/API/规则
- SoulService L2 层 4 个方法：从硬编码占位符 → 元镜动态读取
- `chatExecute` System Prompt 自动注入元镜上下文
- 元镜就绪时，Agent 直接了解系统结构，不再盲猜

### 3. ERDL 运行时字段补全（`a0fa92e`）
- `erdl_crud update` 现在可以处理未在 `.erdl` 文件中声明的字段
- 遇未知字段时，查询 `INFORMATION_SCHEMA.COLUMNS` 动态发现
- 物理表中存在的列 → 自动注册为 `autoDiscovered` → 执行 UPDATE
- 安全性：仅影响 SET 子句，WHERE 条件仍需 ERDL 预定义字段

### 4. 前端 XML 泄露阻断（`1c4f562`）
- 双防线架构：前端 `renderMarkdown` + 后端 `chatExecute` 返回前清理
- 过滤覆盖：invoke / function / parameter / tool_call / thought 标签
- Agent 工具调用 XML 不再泄露到用户可见界面

---

## 🟡 P1 修复（3 项）

### 5. Agent 变更自动提交/编译/测试（`0ef9fb9`）
- 新增 `auto_commit` 工具，Agent 修改代码后一键提交验证
- 六步自动化：git add → diff → commit → build:backend → test → 反馈结果
- 仅 main Agent（L4）可用

### 6. 长任务超时控制（`a06e0ff`）
- `chatExecute` 新增整体超时控制（5 分钟）
- 无外部 AbortSignal 时自动创建超时 AbortController
- 前端 ReAct Timeline 已提供实时进度展示

### 7. Agent 工作区物理隔离（`6b99d45`）
- 新增 `AGENT_WORKSPACE` 环境变量 → Agent 工具操作指向独立仓库
- 新增 `getAgentWorkspace()` 统一入口
- 开发仓库 + 数据库与生产环境完全隔离

---

## 🔧 基础设施

### 8. 元镜引擎启动路径修复（`3a14b92`）
- `projectRoot` 从 `packages/` 修正为 monorepo 根目录
- 修复启动时 `ENOENT` 错误

### 9. 元镜知识文件兜底检查（`18f0031`）
- `needsRegen` 增加文件实际存在性兜底检查
- manifest 缓存存在但知识文件缺失时 → 强制重新生成
- regenerate 成功后输出实际文件路径，便于排障

### 10. 全局版本号统一（`3e6255c` + `b87cc57` + `9946697` + `981d519`）
- 5 个 `package.json` + `main.ts` + 前端 3 个组件 → 全部同步为 `1.6.0-alpha`
- 版本号收敛为单一来源：`frontend/package.json` → Vite `define` → `__APP_VERSION__`
- 新增 `v1.5.1` tag 对齐 GitHub
