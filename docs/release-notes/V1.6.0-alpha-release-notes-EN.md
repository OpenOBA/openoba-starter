# OpenOBA V1.6.0-alpha Release Notes

**Release Date**: 2026-06-29  
**Baseline**: V1.5.1 (`b4ccbc6`, published to GitHub)  
**Commits**: 13

---

## 🔴 P0 Fixes (4 items)

### 1. Agent Identity Fallback Fix (`06da121`)
- Agent no longer downgrades to `sub`/`L1` when `sys_agent_manifest` table is empty
- Core agents (tanghaoran / main-agent) hardcoded fallback to `main`/`L4`
- Ensures admin tools (file_edit / tsc_check / git_diff) are always available

### 2. Meta-Mirror Context Injection into Agent System Prompt (`ce254c4`)
- New `MirrorKnowledgeReader` — dynamically reads entities / APIs / rules from `knowledge/` 
- SoulService L2 layer: 4 methods replaced from hardcoded placeholders → live mirror reading
- `chatExecute` System Prompt now auto-injects mirror context
- When mirror is ready, Agent knows the system structure upfront — no more blind guessing

### 3. ERDL Runtime Field Discovery (`a0fa92e`)
- `erdl_crud update` now handles fields not declared in `.erdl` files
- Queries `INFORMATION_SCHEMA.COLUMNS` when encountering unknown fields
- Columns existing in the physical table → auto-registered as `autoDiscovered` → UPDATE executes
- Safety: only affects SET clause; WHERE conditions still require ERDL-defined fields

### 4. Frontend XML Leak Prevention (`1c4f562`)
- Dual defense: frontend `renderMarkdown` + backend `chatExecute` cleanup before return
- Filters: invoke / function / parameter / tool_call / thought tags
- Agent tool-calling XML no longer leaks to user-visible UI

## 🟡 P1 Fixes (3 items)

### 5. Agent Auto Commit / Build / Test (`0ef9fb9`)
- New `auto_commit` tool: Agent commits changes with one-click verification
- Six steps: git add → diff → commit → build:backend → test → feedback
- Restricted to main Agent (L4) only

### 6. Long-Running Task Timeout Control (`a06e0ff`)
- `chatExecute` now has overall timeout (5 minutes)
- Auto-creates timeout AbortController when no external signal provided
- Frontend React Timeline already provides real-time progress display

### 7. Agent Workspace Physical Isolation (`6b99d45`)
- New `AGENT_WORKSPACE` environment variable → Agent tools operate on isolated repo
- New `getAgentWorkspace()` unified entry point
- Development repo + database fully isolated from production

## 🔧 Infrastructure

### 8. Meta-Mirror Engine Path Fix (`3a14b92`)
- `projectRoot` corrected from `packages/` → monorepo root
- Fixed `ENOENT` error on startup

### 9. Mirror Knowledge File Fallback Check (`18f0031`)
- `needsRegen` now checks if knowledge files actually exist on disk
- Forces regeneration when manifest cache exists but files are missing
- Regenerate logs output actual file paths for visibility

### 10. Global Version Unification (`3e6255c` + `b87cc57` + `9946697` + `981d519`)
- 5 `package.json` + `main.ts` + 3 frontend components → all synced to `1.6.0-alpha`
- Version number consolidated to single source: `frontend/package.json` → Vite `define` → `__APP_VERSION__`
- Added `v1.5.1` tag aligned with GitHub
